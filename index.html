推荐一部纪录片:最后的棒棒

发表于 2018-08-16 | 分类于 生活记录

开始

这两天下班的路上总是看一部纪录片,最后的棒棒,大概是说一个转业的团级军人体验棒棒的过程.
说起棒棒,估计很多人都不太陌生,这是山城重庆的独有工种,重庆地形上上下下,运输并不方便,只能靠人力来完成物资的运输.隐约记得以前有个电视剧 叫山城棒棒军,不过情节已经不太记得了.不过光想想就知道这件事情的辛苦.
这部纪录片2014年开始拍摄,我只看了三集,却感受到了生活的艰辛和人生的变化无常.
先不说记录片的情节,咱们先想几个事

  1. 谁会做棒棒这个工作,
    我想大概年轻人是不会做的,做棒棒不但没有技术成长,反而随着年龄增长,会越发贬值.
    有点钱的估计也不会做这个,有点本钱卖点东西 ,会点技术的也不会做,
    那只有上了岁数的,手里没有积蓄的,没有任何其他技能的才会做这个工作.
  2. 收入怎么样
    谁会叫棒棒来做工呢,比方说搬家,稍微有点钱的会找搬家公司来做,那就是说棒棒的收入是不及搬家公司的,因为棒棒既没有车,也没有更好的服务,可是普通的搬家公司员工已经收入很低了.
    棒棒这个工作没有任何技术含量,提供的服务也没有任何优势可言,因为他的从业人员就能看出来了.
  3. 为什么不做其他的工作
    这个感觉已经不用回答了,自身的年龄,多年的习惯,改变谈何容易,反过来说 如果能改变 早就改变了

情节

说下情节吧,先贴两个图

为了挣20块钱,老黄徒手掏便池.


老黄49年生人,家里有10多亩地,成了所谓的富农,父亲癫痫而死,母亲养活子女六人.
老黄40岁没钱娶媳妇,和一个寡妇搭伙过日子,一起养活寡妇带来的三个孩子,计划生育时代,老黄又生了自己的一个闺女,可是巨额罚款让老黄远走他乡,打工赚钱.三年后的一个电报他匆匆回家,媳妇已经跟了别人,自己的闺女成了别人的累赘.
老黄做起来了棒棒,养活闺女,没有家庭概念的闺女 在18岁时候就跟网上的一个男人啪啪了.
孩子,房子,老黄的棒棒生涯延续到了现在.

反省

看了三集,自己以后会成为时代的弃儿么?个人在社会面前,渺小的如同蚂蚁,60多岁的人,下无儿女抚养,上无国家福利,自己又是被社会淘汰的人,没有安身立命之术.
可能很多人觉得做点什么不比做棒棒强,可是周围的环境,自己的意识,方方面面限制住了,好像无法挣脱出来.
我们同情他们的同时,真的应该反省自己了.
想起来一句话,
让自己不被淘汰的唯一办法就是提供自己的生产率.

os

发表于 2017-12-27 | 分类于 OS

我本科是在一个很普通的高校学习的,所以操作系统这门课还是偏理论一些,而且理论也讲的不太好,对开源的os也没有过任何的分析,看mit等学校的一些课程,感觉project非常的多,从新来学习下os.
OS需要包括哪些概念呢,我们第一时间会想到的进程,内存管理,文件管理等等.
选择的课程是mit 6.828,实践性非常强的一门课程,各种工具文档也非常的齐全,课程的地址
https://pdos.csail.mit.edu/6.828/2011/schedule.html
开始第一天的学习,先说下我们的环境，这个课程推荐在linux下进行

  1. 6.828选了JOS作为他们的课程大作业,而学习os是选择xv6,类似unix的系统
    最开始就有一篇很经典的论文 The Evolution of the Unix Time-Sharing System,了解unix的基本概念.这篇论文几乎就是一本书了，感觉以后很多unix的书 都是这篇论文的翻版.
    下面对image和process的解释非常的简练。

    An image is a computer execution environment. It includes a memory image, general register values,status of open files, current directory and the like. An image is the current state of a pseudocomputer.
    A process is the execution of an image. While the processor is executing on behalf of a process, the image must reside in main memory; during the execution of other processes it remains in main memory unless the appearance of an active, higherpriority process forces it to be swapped out to the disk
